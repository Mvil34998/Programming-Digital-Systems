# Правила ссылок между документами

## 1. Назначение документа

Документ определяет правила построения ссылок между документами проекта Programming Digital Systems.

Ссылки нужны для того, чтобы пользователь видел происхождение информации, связи между решениями и маршрут движения по базе знаний.

## 2. Главный принцип

Ссылка допускается только в том случае, если связанный документ влияет на содержание текущего документа или помогает понять его место в системе знаний.

Запрещены случайные ссылки без объяснения смысла связи.

## 3. Уровни ссылок

### 3.1. Ссылка на документ

Используется, если нужно указать связанный документ целиком.

Пример:

- `[[Roadmap технических требований]]`

### 3.2. Ссылка на раздел

Используется, если нужно указать конкретную тему внутри документа.

Пример:

- `[[Roadmap технических требований#Требования к данным]]`

### 3.3. Ссылка на смысловой блок

Используется, если нужно сослаться на правило, требование, критерий, определение или вопрос анкеты.

Пример:

- `[[Roadmap технических требований#TR-DATA-001]]`

### 3.4. Ссылка на цитату

Используется, если нужно зафиксировать источник формулировки, принципа или определения.

Цитата должна быть короткой и сопровождаться ссылкой на источник.

### 3.5. Ссылка на диаграмму

Используется, если диаграмма является отдельным смысловым объектом.

Пример:

- `[[Диаграммы системы#DG-SYS-001]]`

## 4. Обязательное описание связи

Ссылка должна объяснять смысл связи.

Неправильно:

- `[[Roadmap проектирования системы]]`

Правильно:

- `[[Roadmap проектирования системы]]`
  - Назначение: определяет сущности, данные, правила, состояния, события, потоки, хранение и ошибки.
  - Использование: применяется до формирования технических требований.
  - Ограничение: не должен содержать выбор библиотек, фреймворков и инструментов реализации.

## 5. Раздел связанных документов

Документ должен иметь раздел `Связанные документы`, если он зависит от других документов или передаёт данные в следующие документы.

Рекомендуемая структура:

```md
## Связанные документы

### Входные документы

- `Документ`
  - Передаёт:
  - Используется для:
  - Ограничение:

### Выходные документы

- `Документ`
  - Получает:
  - Используется для:
  - Ограничение:
```

## 6. Идентификаторы блоков

Для большой базы знаний заголовков недостаточно.

Важные блоки должны иметь идентификаторы:

- `REQ-...` — требование;
- `Q-...` — вопрос анкеты;
- `RULE-...` — правило;
- `DEF-...` — определение;
- `DG-...` — диаграмма;
- `EX-...` — пример;
- `DEC-...` — проектное решение.

Пример:

```md
## REQ-DATA-001. Требование к входным данным
```

## 7. Запрещённые ссылки

Запрещены:

- ссылки без объяснения назначения;
- ссылки ради количества;
- ссылки на документы, которые не влияют на текущий текст;
- ссылки на будущие документы без указания их планируемой роли;
- дублирование одной и той же связи в разных формулировках.

## 8. Связанные документы

- `PROJECT_SCOPE.md` — определяет масштаб системы знаний.
- `docs/01_regulations/Documentation_System_Regulation.md` — определяет общую систему документации.
- `docs/01_regulations/Document_Writing_Rules.md` — определяет стиль изложения.
