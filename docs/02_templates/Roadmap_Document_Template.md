# Шаблон Roadmap-документа

## 1. Назначение шаблона

Шаблон определяет обязательную структуру roadmap-документа проекта Programming Digital Systems.

Roadmap-документ должен вести пользователя по проектному этапу и фиксировать движение от входных условий к выходным результатам.

## 2. Обязательная структура

```md
# Roadmap: Название этапа

## 1. Назначение документа

## 2. Место документа в маршруте разработки

## 3. Входные условия

## 4. Связанные документы

## 5. Основные понятия этапа

## 6. Правила этапа

## 7. Порядок работы

## 8. Диаграммы этапа

## 9. Примеры из разных областей цифровых систем

## 10. Контрольные вопросы

## 11. Критерии завершения

## 12. Выходные данные для следующего этапа

## 13. Открытые вопросы

## 14. История изменений
```

## 3. Место документа в маршруте

Каждый roadmap-документ должен указывать:

- какой этап идёт до него;
- какой этап идёт после него;
- какие данные он получает;
- какие данные он передаёт дальше;
- какие решения нельзя принимать на этом этапе.

## 4. Входные условия

Раздел должен отвечать на вопрос:

> Что должно быть уже определено до начала работы с этим документом?

## 5. Связанные документы

Раздел должен содержать входные и выходные документы.

Пример:

```md
## Связанные документы

### Входные документы

- `Документ`
  - Передаёт:
  - Используется для:
  - Ограничение:

### Выходные документы

- `Документ`
  - Получает:
  - Используется для:
  - Ограничение:
```

## 6. Правила этапа

Правила должны быть сформулированы строго.

Допустимые формы:

- `Необходимо определить...`
- `Запрещено...`
- `Если ..., то ...`
- `Этап считается завершённым, если...`

## 7. Диаграммы этапа

Если этап описывает структуру, связи, поток, состояние или последовательность, документ должен содержать диаграмму.

Тип диаграммы выбирается по правилам `docs/01_regulations/Diagram_Rules.md`.

## 9. Критерии завершения

Roadmap-документ считается завершённым, если:

- заполнены обязательные разделы;
- указаны входные и выходные документы;
- правила сформулированы проверяемо;
- примеры не смешаны с категориями;
- диаграммы соответствуют смыслу информации;
- открытые вопросы вынесены отдельно;
- определены выходные данные для следующего этапа.
