# Шаблон анкеты проектного этапа

## 1. Назначение шаблона

Шаблон определяет обязательную структуру анкеты проекта Programming Digital Systems.

Анкета должна превращать правила roadmap-документа в последовательность вопросов и помогать пользователю двигаться от неопределённой идеи к зафиксированному проектному решению.

## 2. Обязательная структура анкеты

```md
# Анкета: Название этапа

## 1. Назначение анкеты

## 2. Связанный roadmap-документ

## 3. Правила заполнения

## 4. Входные данные для анкеты

## 5. Вопросы анкеты

## 6. Контроль полноты ответов

## 7. Выходные данные анкеты

## 8. Открытые вопросы

## 9. История изменений
```

## 3. Структура вопроса

Каждый вопрос анкеты должен иметь:

- идентификатор вопроса;
- формулировку вопроса;
- пояснение;
- поле для ответа;
- критерий заполнения;
- связанные документы;
- следующий шаг.

Пример:

```md
## Q-TR-DATA-001. Какие входные данные должна принимать система?

### Вопрос

Какие данные система должна принимать на вход?

### Пояснение

Необходимо перечислить типы данных, которые система получает до начала обработки.

### Ответ

```text

```

### Критерий заполнения

Ответ считается заполненным, если для каждого типа входных данных указаны:

- источник;
- формат;
- обязательность;
- ограничение размера;
- правило проверки;
- действие при ошибке.

### Связанные документы

- `01_Roadmap_System_Design.md#Данные`.
- `03_Roadmap_Technical_Requirements.md#Требования к данным`.

### Следующий шаг

Перейти к определению правил проверки входных данных.
```

## 4. Правило проверяемости ответа

Ответ анкеты считается корректным, если его можно использовать для принятия проектного решения или формирования требования.

Неправильно:

> Нужно сделать удобно.

Правильно:

> Пользователь должен иметь возможность выбрать входной Excel-файл через диалог выбора файла.

## 5. Правило открытых вопросов

Если пользователь не может ответить на вопрос, ответ не должен заменяться догадкой.

Неопределённость должна быть вынесена в раздел `Открытые вопросы`.

## 6. Правило связи с roadmap-документом

Каждый вопрос анкеты должен быть связан с правилом или разделом соответствующего roadmap-документа.

Анкета не должна содержать вопросы, которые не влияют на проектное решение.

## 7. Критерии завершения анкеты

Анкета считается завершённой, если:

- все обязательные вопросы заполнены;
- ответы сформулированы конкретно;
- открытые вопросы вынесены отдельно;
- ответы можно использовать для следующего проектного этапа;
- нет смешивания требований, инструментов и реализации;
- есть выходные данные анкеты.
