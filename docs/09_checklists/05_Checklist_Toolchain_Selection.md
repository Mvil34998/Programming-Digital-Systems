# 05. Checklist: Toolchain Selection / Выбор инструментария

## 1. Назначение документа

`05_Checklist_Toolchain_Selection.md` проверяет готовность выбора инструментария перед переходом к архитектуре реализации.

Чек-лист подтверждает, что выбранные инструменты связаны с требованиями, архитектурой системы и критериями выбора.

## 3. Чек-лист готовности

### 3.1. Связь с требованиями

- [ ] Каждый выбранный инструмент связан с требованием или критерием выбора.
- [ ] Для каждого выбранного инструмента указано назначение.
- [ ] Для каждого выбранного инструмента указано, какую проблему он решает.
- [ ] Не выбран инструмент без объяснения необходимости.

### 3.2. Категории инструментария

- [ ] Выбран язык программирования, если требуется реализация кода.
- [ ] Выбрана среда выполнения, если она влияет на работу системы.
- [ ] Выбран способ хранения данных, если данные должны сохраняться.
- [ ] Выбраны форматы файлов, если система читает или создаёт файлы.
- [ ] Выбраны библиотеки, если они нужны для конкретных задач.
- [ ] Выбраны инструменты тестирования.
- [ ] Выбраны инструменты логирования.
- [ ] Выбраны инструменты документации.
- [ ] Выбраны инструменты сборки и развёртывания, если они нужны.

### 3.3. Условные специализированные инструменты

- [ ] GUI-инструменты выбраны только если нужен графический интерфейс.
- [ ] Web-инструменты выбраны только если нужна web-система или web-интерфейс.
- [ ] Embedded-инструменты выбраны только если есть встроенное устройство или микроконтроллер.
- [ ] PLC-инструменты выбраны только если есть система промышленной автоматизации.
- [ ] CNC/CAM-инструменты выбраны только если система связана с NC, CAM, постпроцессорами или станочными данными.
- [ ] Инструменты симуляции выбраны только если требуется проверка поведения без реального объекта.

### 3.4. Совместимость

- [ ] Инструменты совместимы с целевой платформой.
- [ ] Инструменты совместимы между собой.
- [ ] Форматы данных поддерживаются выбранными инструментами.
- [ ] Протоколы обмена поддерживаются выбранными инструментами.
- [ ] Лицензии и ограничения использования проверены.

### 3.5. Поддерживаемость

- [ ] Инструменты имеют понятную документацию.
- [ ] Инструменты можно установить и воспроизвести на рабочей среде.
- [ ] Выбор инструментария не создаёт ненужной сложности.
- [ ] Для критичных инструментов есть план замены или обходной путь.

## 4. Блокирующие условия

Переход к архитектуре реализации запрещён, если:

- инструмент выбран без требования;
- специализированный инструмент выбран без применимости;
- инструменты несовместимы между собой;
- невозможно воспроизвести среду разработки;
- неясно, как выбранный инструмент влияет на структуру реализации.
