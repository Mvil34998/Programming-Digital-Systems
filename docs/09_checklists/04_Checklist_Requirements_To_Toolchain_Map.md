# 04. Checklist: Requirements To Toolchain Map / Связь требований с критериями выбора инструментария

## 1. Назначение документа

`04_Checklist_Requirements_To_Toolchain_Map.md` проверяет готовность перехода от технических требований к критериям выбора инструментария.

Этот этап нужен, чтобы инструменты выбирались не по привычке или вкусу, а по проверяемым требованиям и ограничениям.

## 2. Проверяемые документы

- [[docs/00_maps/04_Requirements_To_Toolchain_Map|Requirements To Toolchain Map]]
  - Передаёт: правила связи требований с критериями выбора инструментария.
  - Используется для: проверки трассировки от требования к инструменту.

- [[docs/03_roadmaps/03_Roadmap_Technical_Requirements|Roadmap: Technical Requirements]]
  - Передаёт: список проверяемых требований.
  - Используется для: определения источников критериев выбора.

- [[docs/03_roadmaps/05_Roadmap_Toolchain_Selection|Roadmap: Toolchain Selection]]
  - Получает: готовые критерии выбора.
  - Используется после: завершения этого чек-листа.

## 3. Чек-лист готовности

### 3.1. Выделение требований, влияющих на инструментарий

- [ ] Определены требования к производительности.
- [ ] Определены требования к платформе выполнения.
- [ ] Определены требования к интерфейсам.
- [ ] Определены требования к хранению данных.
- [ ] Определены требования к форматам файлов.
- [ ] Определены требования к протоколам обмена.
- [ ] Определены требования к надёжности.
- [ ] Определены требования к сопровождению и развитию.

### 3.2. Преобразование требований в критерии

- [ ] Для каждого требования, влияющего на инструмент, создан критерий выбора.
- [ ] Каждый критерий сформулирован проверяемо.
- [ ] Каждый критерий связан с исходным требованием.
- [ ] Каждый критерий не содержит преждевременного выбора конкретного инструмента.

### 3.3. Запрет прямого выбора инструментов

- [ ] Не выбран язык программирования без критериев.
- [ ] Не выбрана база данных без критериев.
- [ ] Не выбран GUI-фреймворк без критериев.
- [ ] Не выбран web-фреймворк без критериев.
- [ ] Не выбран PLC/embedded/CNC-инструмент без условия применимости.
- [ ] Не выбрана библиотека только потому, что она знакома разработчику.

### 3.4. Категоризация критериев

- [ ] Критерии разделены на обязательные и желательные.
- [ ] Критерии разделены по категориям инструментария.
- [ ] Специализированные категории помечены условием применения.
- [ ] Указано, какие критерии блокируют неподходящие инструменты.

### 3.5. Подготовка к выбору инструментария

- [ ] Понятно, какие категории инструментов нужно выбирать.
- [ ] Понятно, какие категории инструментов не нужны для данного проекта.
- [ ] Понятно, какие критерии будут использоваться для сравнения вариантов.
- [ ] Понятно, какие требования нельзя нарушить при выборе инструментов.

## 4. Блокирующие условия

Переход к выбору инструментария запрещён, если:

- требования не преобразованы в критерии;
- критерии не связаны с исходными требованиями;
- инструмент уже выбран до анализа критериев;
- специализированные инструменты выбраны без условия применимости;
- неизвестно, какие требования являются обязательными.
