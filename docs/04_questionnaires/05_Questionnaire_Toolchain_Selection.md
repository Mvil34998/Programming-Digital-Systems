# Questionnaire: Toolchain Selection / Анкета выбора инструментария

## 1. Назначение анкеты

`05_Questionnaire_Toolchain_Selection.md` превращает [[docs/03_roadmaps/05_Roadmap_Toolchain_Selection|Roadmap: Toolchain Selection]] в последовательность вопросов для выбора инструментария конкретной цифровой системы.

Анкета должна помочь пользователю последовательно определить:

- какие категории инструментария действительно нужны проекту;
- какие категории являются базовыми;
- какие категории являются прикладными и применяются только при наличии соответствующего типа системы;
- какие категории являются специализированными и применяются только для embedded, PLC, CNC/CAM, промышленной автоматизации или другого специального контекста;
- требования-источники;
- архитектурные ограничения;
- критерии выбора;
- кандидатов;
- сравнение кандидатов;
- выбранные инструменты;
- ограничения выбранных инструментов;
- влияние выбора на архитектуру реализации;
- открытые вопросы.

Анкета не должна формировать технические требования заново.

## 3. Правила заполнения

- Каждый выбранный инструмент должен быть связан с требованием, архитектурным ограничением или внешним обязательным ограничением.
- Нельзя выбирать инструмент только потому, что он знаком или популярен.
- Нельзя считать все категории инструментов обязательными для любого проекта.
- Для каждой прикладной категории нужно указать, когда она применяется и почему нужна в данном проекте.
- Для каждой специализированной категории нужно указать, относится ли проект к соответствующей области.
- PLC-инструменты рассматриваются только если проект связан с промышленной автоматизацией, PLC, HMI, safety, управлением оборудованием или промышленными протоколами.
- Embedded-инструменты рассматриваются только если проект связан с микроконтроллером, устройством, датчиками, исполнительными механизмами или ограниченными ресурсами.
- CNC/CAM-инструменты рассматриваются только если проект связан с NC-программами, CAM-данными, постпроцессорами, инструментом или станками.
- Если категория не применяется, это нужно явно отметить.
- Если выбор инструмента выявил проблему в требованиях, нужно создать открытый вопрос и вернуться к требованиям.

## 4. Входные данные для анкеты

Перед заполнением анкеты должны быть определены:

- технические требования;
- требования, влияющие на выбор инструментария;
- архитектурные ограничения;
- требования к окружению;
- требования к эксплуатации;
- требования к сопровождению;
- требования к тестируемости;
- внешние обязательные ограничения.

## Q-TOOLS-001. Какой тип системы проектируется?

### Вопрос

К какому типу относится система, для которой выбирается инструментарий?

### Пояснение

Тип системы определяет, какие категории инструментария вообще имеют смысл. Обычная Python-утилита не требует PLC-инструментов. PLC-система не выбирается по тем же правилам, что web-сервис.

### Ответ

```text
Тип системы:

Скрипт / утилита: Да / Нет.
GUI-приложение: Да / Нет.
Web-система или API: Да / Нет.
Embedded-система: Да / Нет.
PLC / промышленная автоматизация: Да / Нет.
CNC/CAM-система: Да / Нет.
Интеграционная система: Да / Нет.
Другое:

Пояснение:

```

### Критерий заполнения

Ответ считается заполненным, если указаны основной тип системы, дополнительные типы и специальные области, которые точно не относятся к проекту.

## 6. Контроль полноты ответов

Анкета считается заполненной достаточно для перехода к архитектуре реализации, если:

- определён тип системы;
- базовые инструменты рассмотрены;
- прикладные категории имеют условия применения;
- специализированные категории имеют условия применения или явно помечены как неприменимые;
- все выбранные инструменты связаны с требованиями или архитектурными ограничениями;
- для важных решений указаны кандидаты;
- для важных решений указаны критерии выбора;
- для выбранных инструментов указаны причины выбора;
- для выбранных инструментов указаны ограничения;
- внешние обязательные ограничения отмечены отдельно;
- открытые вопросы вынесены отдельно;
- выбор инструментария не изменяет технические требования без возврата к ним.

## 7. Выходные данные анкеты

После заполнения анкеты должны быть получены:

- тип системы;
- список выбранных базовых инструментов;
- список выбранных прикладных инструментов;
- список выбранных специализированных инструментов, если они нужны;
- список категорий, которые явно не применяются;
- список выбранных языков программирования;
- список выбранных сред выполнения;
- список выбранных способов хранения;
- список выбранных форматов файлов;
- список выбранных библиотек и фреймворков;
- список выбранных протоколов обмена;
- список выбранных инструментов тестирования;
- список выбранных инструментов логирования и диагностики;
- список выбранных инструментов документации;
- список выбранных инструментов сборки и развёртывания;
- список ограничений выбранных инструментов;
- входные данные для [[docs/04_questionnaires/06_Questionnaire_Implementation_Architecture|Questionnaire: Implementation Architecture]].
