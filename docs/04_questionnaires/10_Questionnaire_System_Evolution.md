# Questionnaire: System Evolution / Анкета развития системы

## 1. Назначение анкеты

`Questionnaire_System_Evolution.md` превращает [[docs/03_roadmaps/Roadmap_System_Evolution|Roadmap: System Evolution]] в последовательность вопросов для анализа и планирования развития цифровой системы.

Анкета помогает определить:

- какой запрос развития поступил;
- почему он нужен;
- кто будет пользоваться новой возможностью;
- какой тип изменения требуется;
- на какие части системы влияет изменение;
- какие требования нужно обновить;
- какую архитектуру нужно изменить;
- нужно ли менять инструментарий;
- какие тесты нужно добавить;
- как изменение повлияет на эксплуатацию и сопровождение;
- сохраняется ли обратная совместимость.

Анкета не должна использоваться для обычного исправления дефектов.

## 2. Связанный roadmap-документ

- [[docs/03_roadmaps/Roadmap_System_Evolution|Roadmap: System Evolution]]
  - Передаёт: порядок анализа развития системы.
  - Используется для: построения вопросов анкеты.
  - Ограничение: анкета фиксирует ответы пользователя, а не заменяет roadmap-документ.

## 3. Правила заполнения

- Каждый запрос развития должен иметь причину.
- Каждый запрос должен иметь пользователя, оператора, внешнюю систему или другой потребитель результата.
- Нельзя добавлять новую функцию без анализа влияния.
- Нельзя менять архитектуру без фиксации архитектурного решения.
- Нельзя менять инструментарий без возврата к выбору инструментария.
- Нельзя скрывать нарушение обратной совместимости.
- Если запрос является дефектом, он должен быть передан в сопровождение.
- Если запрос требует исследования, он должен быть помечен как исследовательский.

## 4. Входные данные для анкеты

Перед заполнением анкеты должны быть доступны:

- текущие требования;
- текущая архитектура системы;
- текущая архитектура реализации;
- выбранный инструментарий;
- результаты эксплуатации;
- результаты сопровождения;
- список ограничений текущей системы;
- список предложений пользователей или операторов.

## 6. Контроль полноты ответов

Анкета считается заполненной, если:

- запрос развития зафиксирован;
- определено, что это развитие, а не сопровождение;
- определён тип изменения;
- выполнен анализ влияния на проектирование системы;
- выполнен анализ влияния на требования;
- выполнен анализ влияния на архитектуру системы;
- определено влияние на инструментарий;
- определено влияние на архитектуру реализации;
- определено влияние на тестирование;
- определено влияние на эксплуатацию;
- проверена обратная совместимость;
- принято решение по запросу.

## 7. Выходные данные анкеты

После заполнения анкеты должны быть получены:

- карточка запроса развития;
- классификация запроса;
- анализ влияния;
- список изменяемых требований;
- список изменяемых архитектурных решений;
- решение по инструментарию;
- список изменений реализации;
- список новых тестов;
- список изменений эксплуатации;
- решение по совместимости и миграции;
- итоговое решение по запросу.
