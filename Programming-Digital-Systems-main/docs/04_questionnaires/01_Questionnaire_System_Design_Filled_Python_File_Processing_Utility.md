# Filled Questionnaire: Python File Processing Utility / Заполненная анкета проектирования системы

## 1. Назначение документа

`01_Questionnaire_System_Design_Filled_Python_File_Processing_Utility.md` показывает тестовое заполнение [[docs/04_questionnaires/01_Questionnaire_System_Design|Questionnaire: System Design]] для учебного примера [[docs/06_examples/Scripts/Python_File_Processing_Utility|Python File Processing Utility]].

Документ нужен для проверки, может ли чат-бот прочитать заполненную `.md` анкету и продолжить совместную работу над проектом без лишних домыслов.

> [!info] Главное
> Этот файл является тестом формата анкеты: факты, ограничения, неизвестные места и следующие действия должны быть понятны из заполненных YAML-блоков.

## 2. Правила чтения для чат-бота

Чат-бот должен считать фактом только то, что указано в полях `facts` и `constraints`.

Чат-бот не должен:

- выбирать инструменты реализации на этапе проектирования системы;
- додумывать недостающие ответы;
- считать пустой список подтверждением отсутствия элемента, если в `unknowns` указана неопределённость;
- переносить архитектурные решения в этап проектирования системы.

Если поле содержит `UNKNOWN` или `PARTIAL`, чат-бот должен задать уточняющий вопрос.
