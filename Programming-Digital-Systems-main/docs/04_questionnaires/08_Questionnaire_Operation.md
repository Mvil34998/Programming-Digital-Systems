# Questionnaire: Operation / Анкета эксплуатации

## 1. Назначение анкеты

`08_Questionnaire_Operation.md` превращает [[docs/03_roadmaps/08_Roadmap_Operation|Roadmap: Operation]] в последовательность вопросов для подготовки системы к реальному использованию.

Анкета помогает определить:

- кто использует систему;
- как система запускается;
- как система останавливается;
- какие рабочие сценарии выполняются;
- какие входные данные используются;
- какие результаты ожидаются;
- какие ошибки видит пользователь или оператор;
- какие действия выполняются при ошибках;
- какие логи и диагностические данные нужны;
- какие ограничения эксплуатации существуют.

Анкета не должна исправлять код, менять требования или добавлять новые функции.

> [!info] Главное
> Анкета собирает проектные ответы и переводит правила roadmap в проверяемые поля.

## 2. Связанный roadmap-документ

- [[docs/03_roadmaps/08_Roadmap_Operation|Roadmap: Operation]]
  - Передаёт: порядок подготовки эксплуатации.
  - Используется для: построения вопросов анкеты.
  - Ограничение: анкета фиксирует ответы пользователя, а не заменяет roadmap-документ.

## 3. Правила заполнения

- Каждый рабочий сценарий должен иметь пользователя или оператора.
- Каждый сценарий должен иметь входные данные и ожидаемый результат.
- Каждая ошибка должна иметь действие пользователя или оператора.
- Эксплуатационные ограничения должны быть указаны явно.
- Нельзя скрывать условия, при которых система может работать неправильно.
- Если эксплуатация выявляет дефект, он передаётся в сопровождение.
- Если эксплуатация требует новой функции, она передаётся в развитие системы.

## 4. Входные данные для анкеты

Перед заполнением анкеты должны быть известны:

- результаты тестирования;
- команды запуска;
- команды остановки;
- рабочие сценарии;
- входные данные;
- ожидаемые результаты;
- ошибки и логи;
- ограничения окружения.

Основные входные документы:

- [[docs/04_questionnaires/07_Questionnaire_Testing|Questionnaire: Testing]].
- [[docs/04_questionnaires/06_Questionnaire_Implementation_Architecture|Questionnaire: Implementation Architecture]].

### Вопрос

### Ответ

```text

Оператор:

Внешняя система:
```

## 6. Контроль полноты ответов

Анкета считается заполненной, если:

- определены пользователи и операторы;
- описан запуск;
- описана остановка;
- описаны рабочие сценарии;
- определены входные данные;
- определены результаты;
- описаны эксплуатационные ошибки;
- описаны действия при ошибках;
- определены логи;
- определены ограничения;
- определено, что передаётся в сопровождение.

## 7. Выходные данные анкеты

После заполнения анкеты должны быть получены:

- эксплуатационные роли;
- инструкция запуска;
- инструкция остановки;
- список рабочих сценариев;
- список входных данных;
- список результатов;
- список эксплуатационных ошибок;
- список действий при ошибках;
- список логов и диагностических данных;
- список ограничений эксплуатации;
- входные данные для [[docs/04_questionnaires/09_Questionnaire_Maintenance|Questionnaire: Maintenance]].
