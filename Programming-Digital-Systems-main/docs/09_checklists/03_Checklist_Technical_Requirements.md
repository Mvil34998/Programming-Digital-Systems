# 03. Checklist: Technical Requirements / Технические требования

## 1. Назначение документа

`03_Checklist_Technical_Requirements.md` проверяет готовность технических требований перед переходом к связи требований с критериями выбора инструментария.

> [!info] Главное
> Чек-лист проверяет готовность результата перед переходом к следующему этапу.

## 3. Чек-лист готовности

### 3.1. Источники требований

- [ ] Требования получены из проектирования системы.
- [ ] Требования получены из архитектуры системы.
- [ ] Ограничения предметной области вынесены отдельно.
- [ ] Внешние обязательные constraints явно отмечены.

### 3.2. Проверяемость

- [ ] Каждое требование можно проверить.
- [ ] Для каждого требования указан критерий успешности.
- [ ] Требования не сформулированы как пожелания.
- [ ] Требования не содержат неопределённых слов без метрики.

### 3.3. Классификация требований

- [ ] Определены функциональные требования.
- [ ] Определены требования к данным.
- [ ] Определены требования к интерфейсам.
- [ ] Определены требования к ошибкам.
- [ ] Определены требования к производительности, если они важны.
- [ ] Определены требования к надёжности, если они важны.
- [ ] Определены требования к безопасности, если они важны.
- [ ] Определены эксплуатационные требования, если система будет использоваться регулярно.

### 3.4. Отделение требований от инструментов

- [ ] Требования не выбирают язык программирования без причины.
- [ ] Требования не выбирают библиотеку без внешнего ограничения.
- [ ] Требования не выбирают базу данных без доказанного constraint.
- [ ] Требования описывают нужное свойство системы, а не предпочтительный инструмент.

### 3.5. Трассируемость

- [ ] Каждое требование связано с причиной появления.
- [ ] Каждое требование связано с проверкой.
- [ ] Критичные требования связаны с рисками.
- [ ] Требования, влияющие на инструментарий, явно отмечены.

### 3.6. Приоритеты

- [ ] Требования разделены на обязательные и желательные.
- [ ] Требования, без которых система не работает, отмечены как блокирующие.
- [ ] Требования, влияющие на архитектуру, выделены отдельно.
- [ ] Требования, влияющие на выбор инструментария, выделены отдельно.

## 4. Блокирующие условия

Переход к выбору критериев инструментария запрещён, если:

- требования невозможно проверить;
- требования смешаны с выбором инструментов;
- неясно, какие требования обязательны;
- неясно, какие требования влияют на инструментарий;
- отсутствуют критерии успешности.
