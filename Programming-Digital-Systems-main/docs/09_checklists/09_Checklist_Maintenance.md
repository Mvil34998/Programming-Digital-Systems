# 09. Checklist: Maintenance / Сопровождение

## 1. Назначение документа

`09_Checklist_Maintenance.md` проверяет готовность системы к сопровождению после начала эксплуатации.

Чек-лист подтверждает, что дефекты, исправления, регрессии, обновления и журнал изменений фиксируются проверяемо.

> [!info] Главное
> Чек-лист проверяет готовность результата перед переходом к следующему этапу.

## 3. Чек-лист готовности

### 3.1. Дефекты

- [ ] Определено, как регистрируются дефекты.
- [ ] Для дефекта фиксируются источник, причина и влияние.
- [ ] Определены уровни критичности дефектов.
- [ ] Определено, кто принимает решение об исправлении.

### 3.2. Исправления

- [ ] Исправление связывается с дефектом.
- [ ] Определён способ проверки исправления.
- [ ] Определены риски регрессии.
- [ ] Обновление не выполняется без проверки критичных сценариев.

### 3.3. Регрессии и обновления

- [ ] Определены регрессионные проверки.
- [ ] Определён порядок выпуска обновлений.
- [ ] Определён порядок отката, если он нужен.
- [ ] Журнал изменений обновляется после изменения системы.

## 4. Блокирующие условия

Переход к сопровождению считается рискованным, если:

- дефекты не регистрируются;
- исправления не проверяются;
- нет регрессионных проверок;
- изменения не фиксируются;
- эксплуатационные ошибки не передаются в сопровождение.
