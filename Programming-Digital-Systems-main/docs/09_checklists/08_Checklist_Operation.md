# 08. Checklist: Operation / Эксплуатация

## 1. Назначение документа

`08_Checklist_Operation.md` проверяет готовность системы к эксплуатации после тестирования.

Чек-лист подтверждает, что запуск, рабочие сценарии, ошибки пользователя, ограничения эксплуатации, логи и инструкции определены.

> [!info] Главное
> Чек-лист проверяет готовность результата перед переходом к следующему этапу.

## 2. Проверяемые документы

- [[docs/03_roadmaps/08_Roadmap_Operation|Roadmap: Operation]]
  - Передаёт: правила подготовки эксплуатации.

- [[docs/04_questionnaires/08_Questionnaire_Operation|Questionnaire: Operation]]
  - Передаёт: ответы по эксплуатации.
  - Используется для: проверки полноты эксплуатационных решений.

- [[docs/09_checklists/07_Checklist_Testing|Checklist: Testing]]

## 3. Чек-лист готовности

### 3.1. Запуск системы

- [ ] Определено, как запускать систему.
- [ ] Определены обязательные входные условия запуска.
- [ ] Определены действия при ошибке запуска.
- [ ] Определено, кто отвечает за запуск.

### 3.2. Рабочие сценарии

- [ ] Определены основные сценарии использования.
- [ ] Определены допустимые действия пользователя или оператора.
- [ ] Определены запрещённые действия.
- [ ] Определены критерии успешного выполнения сценария.

### 3.3. Ошибки и ограничения

- [ ] Определены ошибки пользователя или оператора.
- [ ] Определены сообщения об ошибках.
- [ ] Определены ограничения эксплуатации.
- [ ] Определено, какие ошибки требуют сопровождения.

### 3.4. Логи и результат работы

- [ ] Определено, где фиксируется результат работы.
- [ ] Определено, какие события логируются.
- [ ] Определено, кто анализирует логи.
- [ ] Определён срок хранения эксплуатационных данных, если он нужен.

## 4. Блокирующие условия

Переход к эксплуатации запрещён, если:

- запуск системы не описан;
- рабочие сценарии не определены;
- ошибки пользователя не обработаны;
- результат работы невозможно проверить;
- нет связи с тестированием.

## 5. Результат проверки

- [ ] Этап готов к переходу в [[docs/03_roadmaps/09_Roadmap_Maintenance|Roadmap: Maintenance]].
- [ ] Этап требует доработки.
- [ ] Этап заблокирован открытыми вопросами.

## 6. Следующий шаг

После проверки чек-листа необходимо либо перейти к следующему этапу, либо вернуть невыполненные пункты в открытые вопросы.
