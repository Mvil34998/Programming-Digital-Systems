# 10. Checklist: System Evolution / Развитие системы

## 1. Назначение документа

`10_Checklist_System_Evolution.md` проверяет готовность системы к развитию.

Чек-лист подтверждает, что новые функции, изменения сценариев, новые ограничения и возврат к проектированию системы выполняются контролируемо.

> [!info] Главное
> Чек-лист проверяет готовность результата перед переходом к следующему этапу.

## 2. Проверяемые документы

- [[docs/03_roadmaps/10_Roadmap_System_Evolution|Roadmap: System Evolution]]

- [[docs/04_questionnaires/10_Questionnaire_System_Evolution|Questionnaire: System Evolution]]
  - Передаёт: ответы по развитию системы.
  - Используется для: проверки полноты решений по изменениям.

- [[docs/09_checklists/09_Checklist_Maintenance|Checklist: Maintenance]]

## 3. Чек-лист готовности

### 3.1. Запрос развития

- [ ] Определён источник запроса развития.
- [ ] Описана проблема или новая возможность.
- [ ] Определено, какие пользователи или системы затронуты.
- [ ] Запрос отделён от дефекта сопровождения.

### 3.2. Влияние на проектную модель

- [ ] Проверено влияние на сущности.
- [ ] Проверено влияние на данные.
- [ ] Проверено влияние на правила.
- [ ] Проверено влияние на состояния, события и потоки.
- [ ] Проверено влияние на хранение и ошибки.

### 3.3. Влияние на реализацию

- [ ] Проверено влияние на архитектуру системы.
- [ ] Проверено влияние на технические требования.
- [ ] Проверено влияние на инструментарий.
- [ ] Проверено влияние на архитектуру реализации.
- [ ] Проверено влияние на тестирование и эксплуатацию.

## 4. Блокирующие условия

Развитие системы запрещено начинать как обычную правку кода, если:

- не описан запрос развития;
- неизвестно влияние на проектную модель;
- не обновлены требования;
- не определены проверки;
- изменение ломает существующие сценарии без зафиксированного решения.

## 5. Результат проверки

- [ ] Изменение готово к возврату в [[docs/03_roadmaps/01_Roadmap_System_Design|Roadmap: System Design]].
- [ ] Изменение требует доработки.
- [ ] Изменение заблокировано открытыми вопросами.

## 6. Следующий шаг

После проверки чек-листа необходимо либо перейти к следующему этапу, либо вернуть невыполненные пункты в открытые вопросы.
